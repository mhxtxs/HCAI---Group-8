Explainable flower classification using K-Nearest Neighbours

Model Details

Project: Flower image classifier

Algorithm: K-Nearest Neighbors (KNN)

Task: Multi class classification (flower type)

Input: A flower image processed into numeric features

Output: Predicted flower label

How it decides: Compares the image to similar training samples and votes on the label

Evaluation Data

What we test on: A separate set of flower images not used in training

Why: To evaluate performance on unseen images

Same classes: Test data contains the same flower categories as training

Important: Results are reliable only for images similar to the dataset

Training Data

Data type: Labeled flower images

What it includes: Multiple flower categories with example images per class

Variations: Lighting, angle, background, size

Risk: Classes with fewer images may be predicted less accurately

Intended Use

Main purpose: Identify the type of a flower from an image

Where it is useful:

  • Educational demonstrations of KNN
  • Coursework-based flower recognition tools
  • Identification within dataset conditions

Not intended for:

  • Professional botanical confirmation
  • Unseen or rare flower species
  • Any scenario where a wrong label causes harm

Quantitative Analysis

What affects accuracy most:

  • Choice of k value
  • Quality of extracted image features
  • Class balance within the dataset

Runtime behaviour: Prediction slows as dataset size increases because all samples are compared

Factors

Image factors: Lighting, blur, background clutter, camera distance

Data factors: Samples per class, duplicate or low quality images

Model factors: k value, distance metric, feature scaling

Result: These factors influence which samples are considered closest

Metrics

Main metric: Accuracy (correct predictions over total predictions)

Additional checks:

  • Confusion matrix for misclassification patterns
  • Per-class accuracy when data is imbalanced

Why this matters: High overall accuracy can hide weak performance in specific classes

Ethical Considerations

Privacy: No personal or identity data is used

Fairness risk: Classes with more samples may dominate predictions

Responsible use: Predictions should not be treated as guarantees

Caveats and Recommendations

Caveats:

  • Cannot classify flowers not present in training data
  • Sensitive to lighting and background changes
  • Visually similar flowers may be confused

Recommendations:

  • Increase and balance dataset size
  • Test multiple k values
  • Improve preprocessing to reduce noise
